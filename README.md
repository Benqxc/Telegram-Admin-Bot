# Telegram Admin Bot для Discord

Современный многофункциональный администраторский бот для Discord с поддержкой slash-команд.

## 🚀 Основные возможности

- ✅ **Slash-команды** - все команды работают через `/` и отображаются в меню Discord
- ✅ **Система модерации** - таймауты, баны, кики, предупреждения с авто-действиями
- ✅ **Управление ролями** - выдача, снятие, создание ролей
- ✅ **Автовыдача ролей** - автоматическая выдача ролей новым участникам
- ✅ **Система тикетов** - полноценная система поддержки с категориями и транскриптами
- ✅ **Фильтр слов** - настраиваемый фильтр нецензурных слов
- ✅ **Система уровней/XP** - начисление опыта за сообщения с таблицей лидеров
- ✅ **Система приветствий** - настраиваемые приветствия и прощания
- ✅ **Логирование** - логи модерации, участников и сообщений
- ✅ **Анти-спам защита** - автоматическое обнаружение и наказание за спам
- ✅ **Reaction Roles** - выдача ролей по реакциям
- ✅ **Система опросов** - создание опросов с автоматическими реакциями
- ✅ **Напоминания** - создание напоминаний с отложенной отправкой
- ✅ **Розыгрыши** - система giveaway с автоматическим выбором победителей
- ✅ **Кастомные команды** - создание собственных команд
- ✅ **Система верификации** - верификация новых участников через кнопку
- ✅ **Подтверждения** - подтверждения для опасных действий (бан, кик)
- ✅ **Пагинация** - удобный просмотр длинных списков

## 📋 Команды

### Модерация

- `/timeout <пользователь> <длительность> [причина]` - Выдать таймаут (например: 10m, 1h, 1d)
- `/untimeout <пользователь>` - Снять таймаут
- `/ban <пользователь> [причина]` - Заблокировать пользователя (с подтверждением)
- `/kick <пользователь> [причина]` - Выгнать пользователя (с подтверждением)
- `/warn <пользователь> [причина]` - Выдать предупреждение (авто-действия при 3 и 5 варнах)
- `/unwarn <пользователь> <номер>` - Снять предупреждение
- `/warnings <пользователь>` - Показать предупреждения пользователя
- `/clear <количество>` - Удалить сообщения (1-100)
- `/nick <пользователь> <никнейм>` - Изменить никнейм

### Управление ролями

- `/role_add <пользователь> <роль>` - Выдать роль пользователю
- `/role_remove <пользователь> <роль>` - Снять роль с пользователя
- `/role_info <роль>` - Информация о роли
- `/role_create <название> [цвет]` - Создать новую роль (цвет в HEX: #FF0000)

### Автороли

- `/autorole_add <роль>` - Добавить роль в автовыдачу
- `/autorole_remove <роль>` - Удалить роль из автовыдачи
- `/autorole_list` - Показать список автоматических ролей

### Информация

- `/serverinfo` - Информация о сервере
- `/userinfo [пользователь]` - Информация о пользователе
- `/avatar [пользователь]` - Показать аватар пользователя

### Система тикетов

- `/setup_tickets <канал> [категория] [роль_поддержки]` - Создать панель тикетов
- `/ticket_config [категория] [роль_поддержки]` - Настроить систему тикетов
- `/ticket_stats` - Показать статистику тикетов
- `/ticket_close_all` - Закрыть все открытые тикеты (только администраторы)

**Возможности тикетов:**
- Выбор категории при создании (техподдержка, жалоба, вопрос, другое)
- Кнопка "Принять" для модераторов
- Кнопка "Закрыть" с указанием причины
- Кнопка "Добавить участника" для приглашения в тикет
- Автоматическое создание транскриптов при закрытии
- Отправка транскрипта создателю тикета в ЛС

### Фильтр слов

- `/filter_add <слово>` - Добавить слово в фильтр
- `/filter_remove <слово>` - Удалить слово из фильтра
- `/filter_list` - Показать список фильтруемых слов
- `/filter_toggle` - Включить/выключить фильтр слов

### Система уровней

- `/level [пользователь]` - Показать уровень пользователя
- `/leaderboard` - Таблица лидеров (топ-10)
- `/setlevel <пользователь> <уровень>` - Установить уровень (только администраторы)

**Как работает:**
- Пользователи получают 5-15 XP за каждое сообщение
- При достижении нового уровня отправляется уведомление
- Формула уровня: уровень = √(XP / 100)

### Приветствия и прощания

- `/welcome_setup <канал> <сообщение>` - Настроить приветствия
- `/goodbye_setup <канал> <сообщение>` - Настроить прощания

**Переменные для сообщений:**
- `{user}` - упоминание пользователя
- `{username}` - имя пользователя
- `{server}` - название сервера
- `{member_count}` - количество участников

### Логирование

- `/log_setup <тип> <канал>` - Настроить канал логирования

**Типы логов:**
- `mod_log` - логи модерации (баны, кики, варны, таймауты)
- `member_log` - логи участников (вход/выход, верификация)
- `message_log` - логи сообщений (удаленные, отредактированные)

### Опросы

- `/poll <вопрос> <вариант1> <вариант2> [вариант3] [вариант4] [вариант5]` - Создать опрос

Автоматически добавляются реакции для голосования (1️⃣ 2️⃣ 3️⃣ 4️⃣ 5️⃣)

### Напоминания

- `/remind <время> <сообщение>` - Создать напоминание

**Формат времени:** 10m, 1h, 2d, 1w (минуты, часы, дни, недели)

### Reaction Roles

- `/reactionrole_setup <id_сообщения> <эмодзи> <роль>` - Настроить Reaction Role

Пользователи получают/снимают роль, нажимая на эмодзи под сообщением.

### Розыгрыши (Giveaway)

- `/giveaway_start <длительность> <победителей> <приз>` - Начать розыгрыш
- `/giveaway_end <id_сообщения>` - Завершить розыгрыш досрочно
- `/giveaway_reroll <id_сообщения>` - Перевыбрать победителя

**Пример:** `/giveaway_start 1d 2 Discord Nitro`

### Кастомные команды

- `/customcmd_add <название> <ответ>` - Добавить кастомную команду
- `/customcmd_remove <название>` - Удалить кастомную команду
- `/customcmd_list` - Показать список кастомных команд

Кастомные команды вызываются через `!название`

### Верификация

- `/verification_setup <канал> <роль>` - Настроить систему верификации

Создает панель с кнопкой верификации. Новые участники нажимают кнопку для получения роли.

### Анти-спам

- `/antispam_toggle` - Включить/выключить анти-спам

**Как работает:**
- Обнаруживает 5+ сообщений за 10 секунд
- 1-2 нарушения: предупреждение
- 3 нарушения: таймаут на 10 минут

### Утилиты

- `/say <текст>` - Отправить сообщение от имени бота
- `/msg <пользователь> <сообщение>` - Отправить личное сообщение
- `/me <текст>` - Выделить текст курсивом
- `/shrug [текст]` - Добавить ¯\\_(ツ)_/¯
- `/gif <запрос>` - Поиск GIF (требуется Tenor API)
- `/help` - Показать список команд (с пагинацией)

## ⚙️ Установка и запуск

### 1. Установите зависимости

```bash
pip install -r requirements.txt
```

### 2. Создайте файл `.env`

```env
DISCORD_TOKEN=ваш_токен_бота
```

### 3. Настройте права бота в Discord Developer Portal

**Bot Permissions:**
- ✅ Ban Members
- ✅ Kick Members
- ✅ Moderate Members (для таймаутов)
- ✅ Manage Roles
- ✅ Manage Channels
- ✅ Manage Nicknames
- ✅ Manage Messages
- ✅ Read Messages/View Channels
- ✅ Send Messages
- ✅ Manage Threads
- ✅ Embed Links
- ✅ Attach Files
- ✅ Read Message History
- ✅ Add Reactions

**Privileged Gateway Intents:**
- ✅ Server Members Intent
- ✅ Message Content Intent
- ✅ Presence Intent (опционально)

### 4. Пригласите бота на сервер

Используйте ссылку с правильными scope:

```
https://discord.com/api/oauth2/authorize?client_id=ВАШ_CLIENT_ID&permissions=1099511627782&scope=bot%20applications.commands
```

**Обязательно включите:**
- `bot` scope
- `applications.commands` scope

### 5. Запустите бота

```bash
python bot.py
```

## 🔧 Настройка после запуска

### Базовая настройка модерации

1. Настройте логи модерации:
   ```
   /log_setup mod_log #логи-модерации
   /log_setup member_log #логи-участников
   ```

2. Настройте фильтр слов:
   ```
   /filter_add плохое_слово
   /filter_toggle
   ```

3. Включите анти-спам:
   ```
   /antispam_toggle
   ```

### Настройка тикетов

1. Создайте категорию для тикетов
2. Создайте роль поддержки
3. Настройте систему:
   ```
   /setup_tickets #тикеты Категория-Тикетов @Поддержка
   ```

### Настройка приветствий

```
/welcome_setup #приветствия Добро пожаловать, {user}! Ты {member_count}-й участник {server}!
/goodbye_setup #прощания Прощай, {username}! Надеемся увидеть тебя снова.
```

### Настройка верификации

```
/verification_setup #верификация @Участник
```

### Настройка Reaction Roles

1. Создайте сообщение с ролями
2. Скопируйте ID сообщения (ПКМ → Копировать ID)
3. Настройте роли:
   ```
   /reactionrole_setup 123456789 🎮 @Геймер
   /reactionrole_setup 123456789 🎨 @Художник
   ```

## ❗ Решение проблем

### Команды не отображаются в меню

1. Убедитесь, что бот приглашен с scope `applications.commands`
2. Проверьте логи - должно быть "Синхронизировано X slash-команд"
3. Подождите до 1 часа (Discord кэширует команды)
4. Попробуйте выйти и зайти в Discord

### Бот не реагирует на команды

1. Проверьте, что бот онлайн (зеленый статус)
2. Убедитесь, что у бота есть права "Read Messages" и "Send Messages"
3. Проверьте, что Message Content Intent включен в Developer Portal

### Ошибка прав доступа

1. Проверьте иерархию ролей - роль бота должна быть выше ролей, которыми он управляет
2. Убедитесь, что все необходимые права выданы боту

### Тикеты не создаются

1. Проверьте права "Manage Channels"
2. Убедитесь, что у бота есть доступ к категории тикетов
3. Проверьте, что роль бота выше роли поддержки

### Уровни не начисляются

1. Убедитесь, что Message Content Intent включен
2. Проверьте, что бот видит сообщения в канале
