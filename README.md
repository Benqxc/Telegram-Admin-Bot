# Telegram Admin Bot для Discord

Современный администраторский бот для Discord с поддержкой slash-команд.

## 🚀 Что изменилось

- ✅ **Slash-команды** - все команды теперь работают через `/` и отображаются в меню Discord
- ✅ **Автосинхронизация** - команды автоматически регистрируются при запуске
- ✅ **Улучшенные права** - проверка прав для каждой команды
- ✅ **Автовыдача ролей** - автоматическая выдача ролей новым участникам
- ✅ **Фильтр слов** - автоматическое удаление нецензурных сообщений
- ✅ **Система тикетов** - полноценная система поддержки с приватными каналами

## 📋 Доступные команды

### Модерация
- `/ban <пользователь> [причина]` - Заблокировать пользователя
- `/kick <пользователь> [причина]` - Выгнать пользователя
- `/warn <пользователь> [причина]` - Выдать предупреждение
- `/warnings <пользователь>` - Показать предупреждения пользователя
- `/clear <количество>` - Удалить сообщения (1-100)
- `/nick <пользователь> <никнейм>` - Изменить никнейм

### Автороли
- `/autorole_add <роль>` - Добавить роль в автовыдачу
- `/autorole_remove <роль>` - Удалить роль из автовыдачи
- `/autorole_list` - Показать список автоматических ролей

### Система тикетов
- `/setup_tickets <канал> [категория] [роль_поддержки]` - Создать панель тикетов
- `/ticket_config [категория] [роль_поддержки]` - Настроить систему тикетов
- `/ticket_stats` - Показать статистику тикетов
- `/ticket_close_all` - Закрыть все открытые тикеты (только администраторы)

**Как работает система тикетов:**
1. Администратор создает панель тикетов командой `/setup_tickets`
2. Пользователи нажимают кнопку "Открыть тикет" на панели
3. Автоматически создается приватный канал для тикета
4. Модераторы могут принять тикет кнопкой "Принять"
5. Тикет закрывается кнопкой "Закрыть" с указанием причины
6. Все действия логируются (кто создал, кто принял, кто закрыл, причина)

### Общие команды
- `/say <текст>` - Отправить сообщение от имени бота
- `/msg <пользователь> <сообщение>` - Отправить личное сообщение
- `/me <текст>` - Выделить текст курсивом
- `/shrug [текст]` - Добавить ¯\_(ツ)_/¯
- `/gif <запрос>` - Поиск GIF (требуется Tenor API)

## ⚙️ Установка и запуск

### 1. Установите зависимости
```bash
pip install -r requirements.txt
```

### 2. Создайте файл `.env`
```env
DISCORD_TOKEN=ваш_токен_бота
```

### 3. Настройте права бота в Discord Developer Portal

**Важно!** Бот должен иметь следующие права:

#### Bot Permissions:
- ✅ Ban Members
- ✅ Kick Members
- ✅ Manage Roles
- ✅ Manage Channels (для создания тикет-каналов)
- ✅ Manage Nicknames
- ✅ Manage Messages
- ✅ Read Messages/View Channels
- ✅ Send Messages
- ✅ Manage Threads
- ✅ Embed Links
- ✅ Read Message History

#### Privileged Gateway Intents:
- ✅ Server Members Intent
- ✅ Message Content Intent

### 4. Пригласите бота на сервер

Используйте ссылку с правильными scope:
```
https://discord.com/api/oauth2/authorize?client_id=ВАШ_CLIENT_ID&permissions=1099511627782&scope=bot%20applications.commands
```

**Обязательно включите:**
- `bot` scope
- `applications.commands` scope

### 5. Запустите бота
```bash
python bot.py
```

## 🔧 Настройка фильтра слов

Отредактируйте список `BAD_WORDS` в файле [`bot.py`](bot.py:12):
```python
BAD_WORDS = ['мат1', 'мат2', 'плохое_слово']
```

## 📝 Файлы данных

- `warnings.json` - хранит предупреждения пользователей
- `auto_roles.json` - список ролей для автоматической выдачи
- `tickets.json` - данные о тикетах (открытые, закрытые, логи)
- `ticket_config.json` - конфигурация системы тикетов (категория, роль поддержки)

## ❗ Решение проблем

### Команды не отображаются в меню
1. Убедитесь, что бот приглашен с scope `applications.commands`
2. Проверьте логи - должно быть сообщение "Синхронизировано X slash-команд"
3. Подождите до 1 часа (Discord кэширует команды)
4. Попробуйте выйти и зайти в Discord

### Бот не реагирует на команды
1. Проверьте, что бот онлайн (зеленый статус)
2. Убедитесь, что у бота есть права "Read Messages" и "Send Messages"
3. Проверьте, что Message Content Intent включен в Developer Portal

### Ошибка прав доступа
1. Проверьте иерархию ролей - роль бота должна быть выше ролей, которыми он управляет
2. Убедитесь, что все необходимые права выданы боту
